# Generate Variations: Design Alternatives

When the user wants multiple design options, generate variations on
specific axes — not random alternatives.

## What to Vary
- **Layout**: different arrangements of the same content
- **Visual**: different treatments within the locked direction
- **Interaction**: different ways the same action could work
- **Copy**: different tones for the same message

## What NOT to Vary
- Don't vary the brand direction (that's locked)
- Don't vary the content strategy (that's settled)
- Don't vary accessibility requirements (those are non-negotiable)

## Presenting Variations
For each variation:
- One sentence: what this variation explores
- The artifact (HTML, wireframe, description)
- Trade-offs: what this does well, what it sacrifices

Maximum 3-4 variations. More than that is a failure to commit.
